# 🌐 GitHub Pages 设置指南

## 如何在 GitHub 上预览前端界面

### 步骤 1：进入仓库设置

1. 打开你的 GitHub 仓库页面
2. 点击顶部的 **Settings**（设置）标签

### 步骤 2：找到 Pages 设置

在左侧导航菜单中，向下滚动找到 **Pages** 选项并点击

### 步骤 3：配置部署源

在 **Build and deployment** 部分：

1. **Source（源）**: 选择 `Deploy from a branch`
2. **Branch（分支）**: 
   - 选择 `copilot/optimize-frontend-interface`（当前分支）
   - 或者在 PR 合并后选择 `main`
3. **Folder（文件夹）**: 选择 `/docs`
4. 点击 **Save**（保存）按钮

### 步骤 4：等待部署

- GitHub 会开始自动部署，通常需要 1-2 分钟
- 部署成功后，页面顶部会显示网站地址
- 你的预览地址将是：`https://g061206.github.io/cinema/`

## 🎨 预览功能

### ✅ 可用功能

- **完整 UI 展示**：Material Design 风格界面
- **主题切换**：浅色/深色模式切换
- **响应式设计**：在不同设备上测试布局
- **交互动画**：按钮、卡片、输入框的交互效果
- **聊天界面演示**：本地模拟的聊天功能
- **弹幕演示**：发送消息时显示弹幕效果

### ⚠️ 限制说明

以下功能需要完整部署（包含后端服务）：

- ❌ 实时视频播放（需要 SRS 流媒体服务器）
- ❌ 真实的 WebSocket 聊天（需要 Node.js 后端）
- ❌ 多用户实时通信
- ❌ 在线人数统计（预览版使用随机数模拟）

## 🚀 部署完整版本

如需完整功能，请参考主 README.md 中的部署指南：

- 使用 Docker Compose 一键部署
- 或手动部署 SRS + Node.js 服务器

## 📱 在不同设备上测试

访问 GitHub Pages 预览后，你可以：

1. **桌面端**：直接在浏览器中查看
2. **移动端**：
   - 打开浏览器开发者工具（F12）
   - 点击设备工具栏图标
   - 选择不同设备尺寸测试响应式设计
3. **真实移动设备**：直接在手机/平板浏览器中打开链接
